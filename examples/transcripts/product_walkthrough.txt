Host: This walkthrough shows the meeting intelligence pipeline from source capture to reviewed notes.

Host: First, the command layer receives a request and creates a working folder for the recording or transcript. Metadata is written beside the transcript so later steps can see which model produced each artifact.

Host: Second, the transcript is chunked before summarization. Chunking keeps long source material inside the model context window and makes retries cheaper when one request fails.

Host: Third, each chunk produces a note sheet with overview, key facts, decisions, quotes, and action items. The final renderer merges those note sheets locally so the last step is deterministic.
